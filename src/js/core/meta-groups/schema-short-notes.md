   * `schema-documentation.md` defines the following validation types: string, enum, url, integer, color, and  
     object, along with the allowed properties for each type.

   * `resources/schema.json` uses the following validation types:
       * url (for res_canonical, res_shortlink, res_preconnect, res_preconnect_crossorigin, res_dnsPrefetch)   
       * string (for res_manifest)
       * object (for res_alternates)
